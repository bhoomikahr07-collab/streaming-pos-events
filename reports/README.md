# Reports

`rolling_metrics_report.md` is real, generated output — not a mock-up.

It comes from `generate_report.py`, which runs the actual functions in
`src/cdc_logic.py` (`resolve_latest_state`, `compute_windowed_aggregates`)
against events generated by `src/event_producer.py`. These are the same
functions covered by `tests/test_cdc_logic.py`.

**What this does cover**: the correctness of the CDC-resolution and
windowing math — the part that's easy to get subtly wrong (e.g.
double-counting a corrected transaction).

**What this doesn't cover**: actually running Kafka + Spark Structured
Streaming end-to-end. `src/streaming_pipeline.py` is real PySpark code
implementing the same logic natively (window functions + Delta `MERGE` +
`groupBy(window(...))`), but this sandbox has no Spark/Kafka runtime to
execute it against. It's syntax-checked, not run.

To regenerate this report:

```bash
python3 src/event_producer.py
python3 reports/generate_report.py
```

To run the real Spark pipeline once you have PySpark + delta-spark installed:

```bash
pip install -r requirements.txt
python src/event_producer.py                    # writes data/stream_input/
python src/streaming_pipeline.py --env local     # Ctrl+C after a batch or two — it's a streaming job
```
